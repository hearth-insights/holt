    - FD 3: Progress updates, noisy tools (`mvn build >&3`)

## Success Criteria

**For Core Project:**
- Any developer reading `design/agent-pup.md` understands the 3-channel model immediately
- Any user can find and use `holt logs` command easily
- Error messages reference AGENT_LOGGING_GUIDE.md and suggest using `>&3`
- Pup validates stdout and provides helpful errors

**For Independent Clan:**
- A user with NO source code access can:
  - Create an agent that uses the 3-channel model correctly
  - Run noisy tools without corrupting JSON: `mvn build >&3`
  - Debug their agent using `holt logs`
  - Understand why FD 3 exists (keeps stdout clean)
  - Understand all common mistakes and fixes

**For Debugging:**
- Developers can optionally retain worker containers for post-mortem analysis
- `holt up` warns when container retention is enabled
- `holt down` aggressively cleans up all containers (including retained)
