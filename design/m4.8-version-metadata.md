# **Feature design: M4.8 Version Metadata**

**Purpose**: Design for adding standard version metadata to all Holt binaries.
**Scope**: All Holt binaries (`holt`, `holt-orchestrator`, `holt-pup`).
**Estimated tokens**: ~1,500 tokens
**Read when**: Implementing versioning or verifying binary provenance.

Associated phase: Heartbeat (Retroactive improvement)
Status: Draft

***Template purpose:*** *This document is a blueprint for adding version metadata to ensure all Holt binaries can report their identity, version, and build time.*

## **1. The 'why': goal and success criteria**

### **1.1. Goal statement**
Embed git hash, release tag, and build timestamp into all Holt binaries (`holt`, `holt-orchestrator`, `holt-pup`) and expose this via a standard `version` command/flag to enable compatibility verification.

### **1.2. User story**
As an operator or automated system, I want to run any Holt binary with a `version` argument to verify its exact version, commit hash, and build time, so that I can ensure compatibility between the CLI, Orchestrator, and Agents.

### **1.3. Success criteria**
*   Running `holt version` (or `holt --version`) displays version, commit, and date.
*   Running `holt-orchestrator --version` displays version, commit, and date.
*   Running `holt-pup --version` displays version, commit, and date.
*   All binaries built from the same commit/build invocation share the exact same timestamp.
*   The `Makefile` injects this metadata automatically during builds.

### **1.4. Non-goals**
*   Automatic version enforcement (preventing mismatched binaries from running) is a future goal, not part of this specific milestone.
*   Changing the release tagging process itself.

## **2. The 'what': component impact analysis**

### **2.1. Blackboard changes**
No changes.

### **2.2. Orchestrator changes**
*   **New/modified logic**: `cmd/orchestrator/main.go` must handle a `--version` flag before starting the main engine.
*   **New/modified configurations**: None.

### **2.3. Agent pup changes**
*   **New/modified logic**: `cmd/pup/main.go` must handle a `--version` flag before starting.

### **2.4. CLI changes**
*   **New/modified commands**: The existing `version` command in `holt` CLI will be refactored to use the shared version package.
*   **Changes to user output**: Standardized output format across all binaries.

### **2.5. Shared Code (New)**
*   **`pkg/version`**: A new package to hold the build-time variables (`Version`, `Commit`, `Date`). This allows `ldflags` to target a single canonical location.

## **3. The 'how': implementation & testing plan**

### **3.1. Key design decisions & risks**
*   **Shared Package**: We will use a `pkg/version` package with exported variables. The `Makefile` will use `-ldflags "-X github.com/dyluth/holt/pkg/version.Version=..."` to set these. This is cleaner than setting `main.version` in multiple `main` packages.
*   **Timestamp Consistency**: The `Makefile` already calculates `BUILD_DATE` once per invocation. This ensures that `make build-all` produces binaries with identical timestamps, which is a requirement.

### **3.2. Implementation steps**
1.  **[Shared]** Create `pkg/version/version.go` with variables `Version`, `Commit`, `Date` (defaulting to "dev", "none", "unknown").
2.  **[Build]** Update `Makefile` to:
    *   Define `LDFLAGS` targeting `github.com/dyluth/holt/pkg/version`.
    *   Apply `$(LDFLAGS)` to `build-orchestrator`, `build-pup`, and all cross-compilation targets.
3.  **[CLI]** Refactor `cmd/holt/main.go` and `cmd/holt/commands` to use `pkg/version`.
4.  **[Orchestrator]** Update `cmd/orchestrator/main.go` to check for `--version` flag, print info using `pkg/version`, and exit.
5.  **[Pup]** Update `cmd/pup/main.go` to check for `--version` flag, print info using `pkg/version`, and exit.

### **3.3. Performance & resource considerations**
Negligible impact.

### **3.4. Testing strategy**
*   **Unit tests**: None required for simple variable holding.
*   **Manual Verification**: Run `make build-all` and check `./bin/holt version`, `./bin/holt-orchestrator --version`, `./bin/holt-pup --version`. Verify timestamps match.
