# **Feature design: System Integrity & Configuration Ledger**

**Purpose**: Cryptographically bind workflow artefacts to the system configuration state that produced them
**Scope**: Phase 4 deliverable - Configuration drift detection for regulated industries
**Estimated tokens**: ~9,000 tokens

Associated phase: Human-in-the-Loop
Status: Design Draft (2025-11-27)

**Implementation Prerequisites**: M4.6 Verifiable Artefact Ledger (hash-based Merkle DAG)

***Template purpose:*** This document specifies the "System Spine" architecture - a mechanism to cryptographically bind every workflow artefact to the exact system configuration (holt.yml + git commit + agent images) that was active at creation time. This closes the "Configuration Drift" audit gap identified in M4.6, enabling compliance officers to prove not just *what* happened, but *which version of the system* produced each decision.

---

## **1. The 'why': goal and success criteria**

### **1.1. Goal statement**

Extend Holt's Merkle DAG with a "System Spine" - a cryptographically verifiable chain of SystemManifest artefacts that anchor all workflow artefacts to the specific system configuration (holt.yml, git commit, agent versions) active at creation time, enabling auditors to detect configuration drift and prove regulatory compliance.

### **1.2. User story**

As a compliance officer investigating an AI-assisted financial decision made 6 months ago, I need cryptographic proof of which version of the system configuration produced that decision. When I inspect the workflow DAG, I must be able to trace any artefact back to a SystemManifest that immutably records the exact holt.yml configuration file, git commit hash, and agent container versions that were active. If the system configuration changed mid-workflow (e.g., orchestrator restart with updated config), the DAG must clearly show this transition, and I must be able to verify that new artefacts correctly reference the new configuration state. Without this, I cannot satisfy regulatory requirements that demand "complete system provenance" for AI-assisted decisions.

### **1.3. Success criteria**

1. **Startup identity verification**: On orchestrator startup, the system computes a cryptographic identity (SHA-256 of holt.yml + git HEAD commit), queries Redis for the latest SystemManifest, and performs drift detection. If identity matches, the orchestrator reuses the existing manifest ID. If mismatch is detected, it creates a NEW SystemManifest artefact (version++, parent=previous manifest) and stores the new ID in `holt:{instance}:active_manifest`.

2. **Root artefact anchoring**: The CLI `holt forage` command fetches `holt:{instance}:active_manifest`, validates it exists (fails startup if missing), and sets `ParentHashes: [active_manifest_id]` on the GoalDefined artefact. The orchestrator's topology validation enforces this: root artefacts (ProducedByRole="user"/"cli") MUST have exactly one parent hash matching the active manifest, or topology violation is triggered.

3. **Pluggable identity strategy**: The orchestrator supports two identity computation strategies: (A) Local/Default - computes `SHA256(holt.yml)` and runs `git rev-parse HEAD`, failing startup if git repo is invalid; (B) External/Enterprise - reads identity payload from file path specified in `HOLT_MANIFEST_PATH` env var (validates JSON syntax only, opaque schema). Strategy is selected via `HOLT_IDENTITY_SOURCE` env var (defaults to "local").

4. **E2E configuration drift test**: A test starts orchestrator with config v1, creates workflow, stops orchestrator, modifies holt.yml, restarts orchestrator. The test verifies: (1) new SystemManifest created (version=2, parent=v1 manifest hash), (2) new active_manifest points to v2, (3) new GoalDefined artefacts reference v2 manifest, (4) old artefacts still reference v1 manifest (immutable history), (5) `holt hoard` displays both manifests in chronological order.

### **1.4. Non-goals**

- **Agent script content tracking**: The manifest records agent *image IDs* (Docker SHA256), not the script source code. Script versioning is assumed to be managed via git and Docker build process.
- **Runtime agent binary verification**: We do not verify that the running agent container matches the manifest's image ID. This is deferred to container orchestration platforms (Kubernetes admission controllers, etc.).
- **Manifest signing**: Like M4.6, this milestone provides hash-based integrity, not digital signatures. Signing is future work.
- **Retroactive manifest attachment**: SystemManifest is created at orchestrator startup. Artefacts created before this feature was deployed cannot be retroactively anchored.

```go
package blackboard

import (
    "crypto/sha256"
    "encoding/hex"
    "encoding/json"
    "fmt"
    "os"
    "os/exec"
)

// SystemIdentity represents the cryptographic identity of a Holt instance.
type SystemIdentity struct {
    Strategy      string `json:"strategy"`        // "local" or "external"
    ConfigHash    string `json:"config_hash"`     // SHA-256 of holt.yml (local only)
    GitCommit     string `json:"git_commit"`      // Git HEAD commit (local only)
    ComputedAtMs  int64  `json:"computed_at_ms"`  // Timestamp
    ExternalData  string `json:"external_data,omitempty"` // Raw JSON from external file
}

// IdentityProvider is an interface for computing system identity.
type IdentityProvider interface {
    ComputeIdentity() (*SystemIdentity, error)
    IdentityHash() (string, error) // Returns SHA-256 hash of the identity
}

// LocalIdentityProvider computes identity from holt.yml and git.
type LocalIdentityProvider struct {
    ConfigPath string // Path to holt.yml
    WorkspaceRoot string // Git repository root
}

func (p *LocalIdentityProvider) ComputeIdentity() (*SystemIdentity, error) {
    // Step 1: Read and hash holt.yml
    configBytes, err := os.ReadFile(p.ConfigPath)
    if err != nil {
        return nil, fmt.Errorf("failed to read config file: %w", err)
    }

    configHashBytes := sha256.Sum256(configBytes)
    configHash := "sha256:" + hex.EncodeToString(configHashBytes[:])

    // Step 2: Get git HEAD commit (MUST succeed - fail fast if not a git repo)
    cmd := exec.Command("git", "rev-parse", "HEAD")
    cmd.Dir = p.WorkspaceRoot
    output, err := cmd.Output()
    if err != nil {
        return nil, fmt.Errorf("failed to get git commit (ensure workspace is a valid git repository): %w", err)
    }
    gitCommit := string(bytes.TrimSpace(output))

    return &SystemIdentity{
        Strategy:     "local",
        ConfigHash:   configHash,
        GitCommit:    gitCommit,
        ComputedAtMs: time.Now().UnixMilli(),
    }, nil
}

func (p *LocalIdentityProvider) IdentityHash() (string, error) {
    identity, err := p.ComputeIdentity()
    if err != nil {
        return "", err
    }

    // Hash the canonical JSON representation
    canonicalBytes, err := canonicaljson.Marshal(identity)
    if err != nil {
        return "", fmt.Errorf("failed to canonicalize identity: %w", err)
    }

    hashBytes := sha256.Sum256(canonicalBytes)
    return hex.EncodeToString(hashBytes[:]), nil
}

// ExternalIdentityProvider reads identity from external file.
type ExternalIdentityProvider struct {
    ManifestPath string // Path to external manifest file
}

func (p *ExternalIdentityProvider) ComputeIdentity() (*SystemIdentity, error) {
    // Read file
    manifestBytes, err := os.ReadFile(p.ManifestPath)
    if err != nil {
        return nil, fmt.Errorf("failed to read external manifest: %w", err)
    }

    // Validate it's valid JSON (but don't enforce schema)
    var rawData interface{}
    if err := json.Unmarshal(manifestBytes, &rawData); err != nil {
        return nil, fmt.Errorf("external manifest is not valid JSON: %w", err)
    }

    return &SystemIdentity{
        Strategy:     "external",
        ExternalData: string(manifestBytes),
        ComputedAtMs: time.Now().UnixMilli(),
    }, nil
}

func (p *ExternalIdentityProvider) IdentityHash() (string, error) {
    identity, err := p.ComputeIdentity()
    if err != nil {
        return "", err
    }

    // Hash the raw external data (opaque to us)
    hashBytes := sha256.Sum256([]byte(identity.ExternalData))
    return hex.EncodeToString(hashBytes[:]), nil
}

// NewIdentityProvider creates the appropriate provider based on environment.
func NewIdentityProvider(configPath, workspaceRoot string) (IdentityProvider, error) {
    source := os.Getenv("HOLT_IDENTITY_SOURCE")
    if source == "" {
        source = "local" // Default
    }

    switch source {
    case "local":
        return &LocalIdentityProvider{
            ConfigPath:    configPath,
            WorkspaceRoot: workspaceRoot,
        }, nil

    case "external":
        manifestPath := os.Getenv("HOLT_MANIFEST_PATH")
        if manifestPath == "" {
            return nil, fmt.Errorf("HOLT_IDENTITY_SOURCE=external requires HOLT_MANIFEST_PATH env var")
        }
        return &ExternalIdentityProvider{
            ManifestPath: manifestPath,
        }, nil

    default:
        return nil, fmt.Errorf("unknown HOLT_IDENTITY_SOURCE: %q (must be 'local' or 'external')", source)
    }
}
```

**New keys**:
- **`holt:{instance}:active_manifest`** - STRING storing the SHA-256 hash ID of the current SystemManifest

```go
package identity

import (
    "fmt"
    "time"

    "github.com/hearth-insights/holt/pkg/blackboard"
)

// SpineManager manages the System Spine lifecycle.
type SpineManager struct {
    blackboard       *blackboard.Client
    identityProvider blackboard.IdentityProvider
    instanceName     string
    spineThreadID    string // Shared LogicalThreadID for this instance's spine
}

// InitializeSpine performs startup drift detection and manifest management.
func (sm *SpineManager) InitializeSpine() (activeManifestID string, err error) {
    // Step 1: Compute current system identity
    currentIdentity, err := sm.identityProvider.ComputeIdentity()
    if err != nil {
        return "", fmt.Errorf("failed to compute system identity: %w", err)
    }

    currentIdentityHash, err := sm.identityProvider.IdentityHash()
    if err != nil {
        return "", fmt.Errorf("failed to hash identity: %w", err)
    }

    log.Info("Computed system identity", "hash", currentIdentityHash[:16]+"...")

    // Step 2: Query for latest SystemManifest in this instance's spine
    latestManifest, err := sm.fetchLatestManifest()
    if err == ErrNoManifest {
        // First startup - create initial manifest
        log.Info("No existing SystemManifest found - creating initial manifest")
        return sm.createManifest(currentIdentity, currentIdentityHash, nil, 1)
    }
    if err != nil {
        return "", fmt.Errorf("failed to query latest manifest: %w", err)
    }

    // Step 3: Drift detection - compare identity hashes
    latestIdentityHash, err := sm.extractIdentityHash(latestManifest)
    if err != nil {
        return "", fmt.Errorf("failed to extract identity from latest manifest: %w", err)
    }

    if currentIdentityHash == latestIdentityHash {
        // No drift - reuse existing manifest
        log.Info("System identity unchanged - reusing manifest",
            "manifest_id", latestManifest.ID[:16]+"...",
            "version", latestManifest.Header.Version)

        // Update active_manifest pointer (idempotent)
        if err := sm.setActiveManifest(latestManifest.ID); err != nil {
            return "", err
        }

        return latestManifest.ID, nil
    }

    // Step 4: Drift detected - create new manifest
    log.Warn("Configuration drift detected - creating new SystemManifest",
        "old_hash", latestIdentityHash[:16]+"...",
        "new_hash", currentIdentityHash[:16]+"...",
        "old_version", latestManifest.Header.Version)

    return sm.createManifest(
        currentIdentity,
        currentIdentityHash,
        []string{latestManifest.ID}, // Parent = previous manifest
        latestManifest.Header.Version+1,
    )
}

func (sm *SpineManager) createManifest(identity *SystemIdentity, identityHash string, parentHashes []string, version int) (string, error) {
    // Marshal identity to JSON payload
    payloadBytes, err := json.Marshal(identity)
    if err != nil {
        return "", fmt.Errorf("failed to marshal identity: %w", err)
    }

    // Create VerifiableArtefact
    manifest := &blackboard.VerifiableArtefact{
        Header: blackboard.ArtefactHeader{
            ParentHashes:    parentHashes,
            LogicalThreadID: sm.spineThreadID, // Shared thread for this instance
            Version:         version,
            CreatedAtMs:     time.Now().UnixMilli(),
            ProducedByRole:  "orchestrator",
            StructuralType:  blackboard.StructuralTypeSystemManifest,
            Type:            "SystemConfig",
            ClaimID:         "", // No claim for system artefacts
        },
        Payload: blackboard.ArtefactPayload{
            Content: string(payloadBytes),
        },
    }

    // Compute hash
    manifestID, err := blackboard.ComputeArtefactHash(manifest)
    if err != nil {
        return "", fmt.Errorf("failed to compute manifest hash: %w", err)
    }
    manifest.ID = manifestID

    // Write to blackboard
    if err := sm.blackboard.WriteArtefact(manifest); err != nil {
        return "", fmt.Errorf("failed to write manifest: %w", err)
    }

    // Update spine ZSET
    if err := sm.addToSpine(manifestID, version); err != nil {
        return "", fmt.Errorf("failed to update spine: %w", err)
    }

    // Update active_manifest pointer
    if err := sm.setActiveManifest(manifestID); err != nil {
        return "", err
    }

    log.Info("Created new SystemManifest",
        "manifest_id", manifestID[:16]+"...",
        "version", version,
        "identity_hash", identityHash[:16]+"...")

    return manifestID, nil
}

func (sm *SpineManager) fetchLatestManifest() (*blackboard.VerifiableArtefact, error) {
    // Query spine ZSET for highest version
    key := fmt.Sprintf("holt:%s:system_spine:%s", sm.instanceName, sm.spineThreadID)

    // ZREVRANGE returns members in descending order by score (version)
    results, err := sm.blackboard.Redis().ZRevRangeWithScores(ctx, key, 0, 0).Result()
    if err == redis.Nil || len(results) == 0 {
        return nil, ErrNoManifest
    }
    if err != nil {
        return nil, err
    }

    manifestID := results[0].Member.(string)
    return sm.blackboard.FetchArtefact(manifestID)
}

func (sm *SpineManager) setActiveManifest(manifestID string) error {
    key := fmt.Sprintf("holt:%s:active_manifest", sm.instanceName)
    return sm.blackboard.Redis().Set(ctx, key, manifestID, 0).Err() // No TTL
}

var ErrNoManifest = fmt.Errorf("no SystemManifest found")
```

```go
func (o *Orchestrator) validateTopology(a *blackboard.VerifiableArtefact) error {
    // ... existing topology validation from M4.6 ...

    // NEW RULE: Root artefact anchor validation
    if a.Header.ProducedByRole == "user" || a.Header.ProducedByRole == "cli" {
        // Root artefacts MUST have exactly one parent: the active SystemManifest
        if len(a.Header.ParentHashes) != 1 {
            alert := createTopologyAlert(a, "root_missing_manifest_anchor")
            triggerGlobalLockdown(alert)
            return fmt.Errorf("root artefact must have exactly one parent (active manifest), got %d", len(a.Header.ParentHashes))
        }

        // Verify parent is the active manifest
        if a.Header.ParentHashes[0] != o.activeManifestID {
            // SOFT CHECK: Allow old manifest if it exists in history (continuity rule)
            exists, err := o.blackboard.ArtefactExists(a.Header.ParentHashes[0])
            if err != nil {
                return fmt.Errorf("failed to check parent manifest existence: %w", err)
            }

            if !exists {
                // Invalid manifest reference
                alert := createTopologyAlert(a, "root_invalid_manifest_reference")
                triggerGlobalLockdown(alert)
                return fmt.Errorf("root artefact references non-existent manifest: %s", a.Header.ParentHashes[0])
            }

            // Valid historical manifest - log warning but accept
            log.Warn("Root artefact anchored to old manifest (config drift race condition)",
                "artefact_id", a.ID[:16]+"...",
                "expected_manifest", o.activeManifestID[:16]+"...",
                "actual_manifest", a.Header.ParentHashes[0][:16]+"...")
        }

        // Valid root artefact - skip further topology checks
        goto hashVerification
    }

    // ... continue with existing agent topology validation from M4.6 ...
```

```go
func (c *SpineCommand) Run(args []string) error {
    // Fetch active manifest
    activeManifestKey := fmt.Sprintf("holt:%s:active_manifest", c.instanceName)
    activeManifestID, err := c.redis.Get(ctx, activeManifestKey).Result()
    if err == redis.Nil {
        fmt.Println("No active manifest found (orchestrator not initialized)")
        return nil
    }
    if err != nil {
        return fmt.Errorf("failed to fetch active manifest: %w", err)
    }

    // Fetch active manifest artefact
    activeManifest, err := c.blackboard.FetchArtefact(activeManifestID)
    if err != nil {
        return fmt.Errorf("failed to fetch manifest artefact: %w", err)
    }

    // Traverse spine backwards via ParentHashes
    manifests := []*blackboard.VerifiableArtefact{activeManifest}
    current := activeManifest

    for len(current.Header.ParentHashes) > 0 {
        parentID := current.Header.ParentHashes[0]
        parent, err := c.blackboard.FetchArtefact(parentID)
        if err != nil {
            log.Warn("Failed to fetch parent manifest", "parent_id", parentID, "err", err)
            break
        }
        manifests = append(manifests, parent)
        current = parent
    }

    // Display in chronological order (oldest first)
    fmt.Println("System Spine History")
    fmt.Println("====================")
    fmt.Println()

    for i := len(manifests) - 1; i >= 0; i-- {
        m := manifests[i]
        isActive := m.ID == activeManifestID

        // Parse identity payload
        var identity map[string]interface{}
        json.Unmarshal([]byte(m.Payload.Content), &identity)

        fmt.Printf("Version %d%s\n", m.Header.Version, activeMarker(isActive))
        fmt.Printf("  Manifest ID: %s\n", m.ID[:16]+"...")
        fmt.Printf("  Created:     %s\n", formatTimestamp(m.Header.CreatedAtMs))

        if strategy, ok := identity["strategy"].(string); ok && strategy == "local" {
            fmt.Printf("  Config Hash: %s\n", truncateHash(identity["config_hash"]))
            fmt.Printf("  Git Commit:  %s\n", identity["git_commit"])
        } else if strategy == "external" {
            fmt.Printf("  Strategy:    External (opaque)\n")
        }

        if i > 0 {
            fmt.Println()
        }
    }

    return nil
}

func activeMarker(isActive bool) string {
    if isActive {
        return " (ACTIVE)"
    }
    return ""
}
```

```go
func (c *VerifyConfigCommand) Run(args []string) error {
    // Fetch specified manifest
    manifest, err := c.blackboard.FetchArtefact(c.manifestID)
    if err != nil {
        return fmt.Errorf("failed to fetch manifest %s: %w", c.manifestID, err)
    }

    // Verify structural type
    if manifest.Header.StructuralType != blackboard.StructuralTypeSystemManifest {
        return fmt.Errorf("artefact is not a SystemManifest (type: %s)", manifest.Header.StructuralType)
    }

    // Parse stored identity
    var storedIdentity blackboard.SystemIdentity
    if err := json.Unmarshal([]byte(manifest.Payload.Content), &storedIdentity); err != nil {
        return fmt.Errorf("failed to parse manifest payload: %w", err)
    }

    fmt.Printf("Verifying SystemManifest %s...\n", c.manifestID[:16]+"...")
    fmt.Println()

    // Strategy-specific verification
    if storedIdentity.Strategy == "local" {
        // Recompute current config hash
        configBytes, err := os.ReadFile(c.configPath)
        if err != nil {
            return fmt.Errorf("failed to read current config: %w", err)
        }

        currentConfigHashBytes := sha256.Sum256(configBytes)
        currentConfigHash := "sha256:" + hex.EncodeToString(currentConfigHashBytes[:])

        // Compare config hashes
        fmt.Printf("Config Hash Verification:\n")
        fmt.Printf("  Stored:  %s\n", storedIdentity.ConfigHash)
        fmt.Printf("  Current: %s\n", currentConfigHash)

        if storedIdentity.ConfigHash == currentConfigHash {
            fmt.Printf("  Status:  ✓ MATCH\n")
        } else {
            fmt.Printf("  Status:  ✗ MISMATCH (config has changed since manifest creation)\n")
        }
        fmt.Println()

        // Verify git commit (informational only - git history may have changed)
        cmd := exec.Command("git", "rev-parse", "HEAD")
        cmd.Dir = c.workspaceRoot
        output, err := cmd.Output()
        currentCommit := strings.TrimSpace(string(output))

        fmt.Printf("Git Commit Verification:\n")
        fmt.Printf("  Stored:  %s\n", storedIdentity.GitCommit)
        if err != nil {
            fmt.Printf("  Current: (git command failed: %v)\n", err)
        } else {
            fmt.Printf("  Current: %s\n", currentCommit)
            if storedIdentity.GitCommit == currentCommit {
                fmt.Printf("  Status:  ✓ MATCH\n")
            } else {
                fmt.Printf("  Status:  ✗ MISMATCH (git HEAD has moved)\n")
            }
        }

    } else if storedIdentity.Strategy == "external" {
        fmt.Printf("Strategy: External (opaque)\n")
        fmt.Printf("  Stored identity is opaque JSON - manual verification required\n")
        fmt.Printf("  Payload: %s\n", storedIdentity.ExternalData)
    }

    fmt.Println()
    fmt.Printf("Manifest Created: %s\n", formatTimestamp(manifest.Header.CreatedAtMs))
    fmt.Printf("Manifest Version: %d\n", manifest.Header.Version)

    return nil
}
```

## **3. The 'how': implementation & testing plan**

### **3.1. Key design decisions & risks**

#### **Decision 1: Single spine thread per instance**

**Rationale**: Each Holt instance has exactly one "System Spine" logical thread. This creates a linear configuration history, avoiding DAG complexity for system-level state.

**Trade-off**: If multiple orchestrators run on the same instance name (misconfiguration), they will conflict on the spine. This is acceptable - instance names MUST be unique per workspace.

#### **Decision 2: Fail fast on missing git repository (Local strategy)**

**Rationale**: Holt is opinionated - it assumes git-native workflows. If the workspace is not a git repo, the audit trail is fundamentally incomplete (CodeCommit artefacts would be meaningless).

**Alternative considered and rejected**: Allowing empty git commit would silently degrade audit quality. Failing startup forces correct environment setup.

#### **Decision 3: Soft enforcement of continuity rule**

**Rationale**: Distributed systems have race conditions. An agent might start processing before orchestrator restart and finish after config change. Blocking that submission would lose work and violate agent statelessness.

**Mechanism**: Log warning but accept artefacts anchored to old (valid) manifests. Auditors can detect drift by inspecting manifest references.

#### **Risk: External manifest file tampering**

**Scenario**: Malicious actor modifies `HOLT_MANIFEST_PATH` file between identity computation and artefact creation.

**Mitigation**: The identity hash is computed once at startup and stored in SystemManifest. Even if file changes, the hash is immutable. Future verification compares against the stored identity, not the file.

**Residual risk**: If the file is tampered with *before* startup, the SystemManifest will reflect the tampered state. This is acceptable - we trust the filesystem security (root/container isolation). Future work: signature verification.

## **4. Principle compliance check**

### **4.1. YAGNI (You ain't gonna need it)**

**New dependencies**: None. Uses existing `crypto/sha256`, `os/exec` for git, and `encoding/json` from stdlib.

**Justification**: Configuration drift detection is a regulatory requirement for banking/defence. Not nice-to-have.

### **4.2. Auditability**

**Enhanced auditability**:
- Every workflow artefact now traces back to exact system configuration (holt.yml + git commit + agent versions)
- Configuration changes create immutable audit trail (SystemManifest chain)
- Third-party auditors can verify configuration provenance using `holt spine`

This elevates auditability from "what happened" to "what happened under which system version".

### **4.3. Small, single-purpose components**

**Component separation maintained**:
- **Blackboard** (`pkg/blackboard/identity.go`): Identity computation logic
- **Orchestrator** (`internal/orchestrator/identity/spine.go`): Spine management
- **CLI** (`cmd/holt/commands/spine.go`, `forage.go`): User interface

No boundary violations. Each component has single responsibility.

### **4.4. Security considerations**

#### **Attack surface analysis**

**Threat 1: Malicious config modification between restarts**
- **Detection**: SystemManifest records exact config hash. Drift is detected and logged.
- **Mitigation**: Config changes are auditable. Investigators can diff manifest payloads.
- **Residual risk**: None if filesystem security is trusted.

**Threat 2: Git commit spoofing**
- **Scenario**: Attacker modifies git history to hide malicious code
- **Mitigation**: Git commit is recorded in SystemManifest. Verifiers can compare against trusted git remote.
- **Residual risk**: High if git repository itself is compromised. Future work: signed commits.

**Threat 3: External manifest tampering (Strategy B)**
- **Scenario**: Attacker modifies `HOLT_MANIFEST_PATH` file
- **Mitigation**: Identity hash computed once at startup. File changes after startup don't affect recorded identity.
- **Residual risk**: If file is tampered with *before* startup, manifest reflects tampered state (acceptable - trust filesystem).

**Threat 4: Active manifest key manipulation**
- **Scenario**: Attacker directly modifies `holt:{instance}:active_manifest` Redis key
- **Mitigation**: M4.6 topology validation would detect invalid manifest reference (orphan block alert)
- **Residual risk**: Low with proper Redis ACLs (future work)

#### **Credential and secret handling**

**No secrets introduced**. Config hash and git commit are non-sensitive metadata.

#### **Container isolation**

**No changes**. SystemManifest creation is orchestrator-side only.

### **4.5. Backward compatibility**

**Breaking changes**: None. This is additive to M4.6.

**Migration path**:
- Existing V2 instances: First restart after upgrade creates initial SystemManifest
- Existing V1 instances: Must upgrade to M4.6 first (clean break already documented)

**Compatibility**: Old artefacts (created before M4.7) have empty ParentHashes for roots. Topology validation should handle gracefully:

```go
// Graceful handling for pre-M4.7 artefacts
if a.Header.ProducedByRole == "user" && len(a.Header.ParentHashes) == 0 {
    log.Warn("Pre-M4.7 root artefact detected (no manifest anchor)", "artefact_id", a.ID)
    // Accept but log for audit awareness
    return nil
}
```

### **4.6. Dependency impact**

#### **Go version**: No change (uses stdlib only)

#### **Redis version**: No change (ZSET available since Redis 1.2)

#### **Docker version**: No change

#### **Git**: **NEW REQUIREMENT** for Local strategy. Must be available in PATH.

**CI/CD impact**: No changes. Git is already required for development.

## **6. Error scenarios & edge cases**

### **6.1. Failure modes**

#### **Scenario 1: Git repository not initialized**

**Trigger**: Orchestrator starts in workspace without `.git` directory (Local strategy)

**Handling**:
```go
// LocalIdentityProvider.ComputeIdentity()
cmd := exec.Command("git", "rev-parse", "HEAD")
output, err := cmd.Output()
if err != nil {
    return nil, fmt.Errorf("failed to get git commit (ensure workspace is a valid git repository): %w", err)
}
```

**Outcome**: Orchestrator fails startup with clear error message. Operator must run `git init && git commit --allow-empty -m "Initial commit"`.

**User guidance**: Add to troubleshooting doc:
```
Error: "failed to get git commit"
Solution: Initialize git repository in workspace:
  $ git init
  $ git commit --allow-empty -m "Initial commit"
  $ holt up
```

#### **Scenario 2: External manifest file missing**

**Trigger**: `HOLT_IDENTITY_SOURCE=external` but `HOLT_MANIFEST_PATH` points to non-existent file

**Handling**:
```go
// ExternalIdentityProvider.ComputeIdentity()
manifestBytes, err := os.ReadFile(p.ManifestPath)
if err != nil {
    return nil, fmt.Errorf("failed to read external manifest: %w", err)
}
```

**Outcome**: Orchestrator fails startup. Operator must create manifest file or fix path.

#### **Scenario 3: Invalid JSON in external manifest**

**Trigger**: `HOLT_MANIFEST_PATH` file contains malformed JSON

**Handling**: JSON syntax validation fails, orchestrator startup blocked with parse error.

**Outcome**: Clear error message with line/column of syntax error.

#### **Scenario 4: Redis unavailable during spine initialization**

**Trigger**: Redis connection fails during `fetchLatestManifest()`

**Handling**: Orchestrator startup fails (existing resilience behavior - fail fast on Redis unavailability).

**Outcome**: Orchestrator exits, health check fails, container restarts.

#### **Scenario 5: Active manifest key exists but artefact deleted**

**Trigger**: Manual Redis manipulation or data corruption

**Handling**:
```go
activeManifestID, _ := redis.Get("holt:instance:active_manifest").Result()
manifest, err := bb.FetchArtefact(activeManifestID)
if err == ErrArtefactNotFound {
    log.Error("Active manifest artefact missing (data corruption detected)")
    return fmt.Errorf("active manifest %s not found in blackboard", activeManifestID)
}
```

**Outcome**: Orchestrator startup fails. Recovery requires Redis restore from backup or manual manifest recreation.

### **6.2. Concurrency considerations**

#### **Race condition: Multiple orchestrators on same instance**

**Scenario**: Misconfiguration - two orchestrator processes start with same instance name

**Analysis**: Both will attempt to create/update `active_manifest` key. Last write wins (Redis SET is atomic).

**Outcome**: Undefined behavior. This is a **critical misconfiguration** detected by existing instance lock mechanism (M1.1). Not specific to M4.7.

**Mitigation**: Instance lock (TTL-based heartbeat) prevents this scenario.

#### **Race condition: Config change during workflow execution**

**Scenario**: Agent is processing claim when orchestrator restarts with new config

**Analysis**:
1. Agent finishes work after restart
2. Submits artefact with old manifest ID as root parent (from context chain)
3. Topology validation: checks if old manifest exists (yes) → logs warning, accepts artefact

**Outcome**: Safe. Continuity rule (soft enforcement) handles this gracefully.

### **6.3. Edge case handling**

#### **Edge case 1: Empty holt.yml file**

**Scenario**: Config file is valid YAML but has no content

**Handling**: Hash is computed on empty bytes → deterministic hash `sha256:e3b0c442...` (SHA-256 of empty string)

**Outcome**: Valid but semantically useless. Orchestrator will fail later when parsing config.

#### **Edge case 2: Git detached HEAD state**

**Scenario**: Git repo is in detached HEAD state (not on a branch)

**Handling**: `git rev-parse HEAD` still returns commit hash

**Outcome**: Valid. Manifest records the commit, even if not on a named branch.

#### **Edge case 3: Very long spine chain (1000+ versions)**

**Scenario**: Instance runs for months with frequent config changes

**Handling**: `fetchLatestManifest()` uses ZREVRANGE 0 0 (O(log N) operation)

**Outcome**: No performance degradation. Redis sorted sets handle large cardinality efficiently.

#### **Edge case 4: Manifest hash collision (theoretical)**

**Scenario**: Two different identities produce same SHA-256 hash

**Handling**: Not handled (same as M4.6). SHA-256 collision resistance is foundational assumption.

**Probability**: 2^-256 (effectively impossible).

### **8.2. Common pitfalls to avoid**

#### **Pitfall 1: Forgetting to handle missing git repo**

#### **Pitfall 2: Comparing identity structs instead of hashes**

**Incorrect**:
```go
if currentIdentity == latestIdentity { // Struct comparison fails!
    // ...
}
```

**Correct**:
```go
currentHash, _ := provider.IdentityHash()
latestHash, _ := extractIdentityHash(latestManifest)
if currentHash == latestHash {
    // ...
}
```

#### **Pitfall 3: Not handling pre-M4.7 root artefacts**

**Incorrect**:
```go
if len(a.Header.ParentHashes) != 1 {
```

**Correct**:

### **9.2. Rollback and disaster recovery**

#### **Rollback procedure**

**Scenario**: Critical bug in spine initialization logic

**Steps**:
1. **Stop orchestrator**: `holt down`
2. **Deploy previous binary version** (before M4.7)
3. **Clear spine keys** (optional - old binary ignores them):
   ```bash
   redis-cli DEL holt:instance:active_manifest
   redis-cli DEL holt:instance:system_spine:*
   ```
4. **Restart orchestrator**: `holt up`

**Data loss**: SystemManifest history is lost. Existing workflow artefacts remain intact (not dependent on manifests).

#### **Disaster recovery: Spine corruption**

**Scenario**: `active_manifest` key points to non-existent artefact

**Recovery**:
1. **Identify latest valid manifest**: `redis-cli ZREVRANGE holt:instance:system_spine:thread-id 0 0`
2. **Reset active_manifest**: `redis-cli SET holt:instance:active_manifest <valid-manifest-id>`
3. **Restart orchestrator**

#### **Feature disable mechanism**

**Not applicable**. System Spine is a core audit feature. Disabling it would break topology validation for root artefacts.

**Graceful degradation**: If spine initialization fails, orchestrator startup fails (fail-fast principle).
